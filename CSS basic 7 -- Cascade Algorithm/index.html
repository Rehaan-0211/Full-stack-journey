Cascade Algorithm

Jokes apart lets discuss how this Cascade Algorithm actually works..it has for points

  1. Position and order of appearance
  2. Specificity
  3. Origin
  4. Importance
Lets discuss about them breifly.

Position

When there are same type of selectors present...then the selector which is present in the end most part of the code is considered for styling as the other selectors are over ridden by the end selectors
Ex:- Class 1 -> Class 2 -> Class 3.
Class 3 color or code will be used.

Specificity

Specificity is considered in the following order

Inline CSS->ID Selectors->Class or Attribute Selector->Element Selector->Universal Selector

Specificity Calculation

  1. Universal Selector > 0
  2. Element selector or pseudo elements > 1
  3. Class/Attribute selector or pseudo classes > 10
  4. ID Selectors > 100
  5. Inline CSS> 1000
  6. Important> 10000
